## 검토 체크리스트

### 1. frontmatter 유효성

| 항목 | 기준 | 심각도 |
|------|------|--------|
| `title` | 존재하는가 | 🚨 필수 |
| `pubDatetime` | ISO 8601 + `+09:00` 형식인가 | 🚨 필수 |
| `description` | 50~160자인가 | 🚨 SEO 직결 |
| `author` | `suko_yasa`인가 | ⚠️ 권장 |
| `tags` | 2~5개인가 | ⚠️ 권장 |
| `ogImage` | 존재하는가, 경로가 유효한가 | ⚠️ SNS 공유에 영향 |
| `draft` | `false`인가 (발행 예정이라면) | ⚠️ 확인 |

### 2. 구조 검토

| 항목 | 기준 | 심각도 |
|------|------|--------|
| 핵심 요약 박스 | 본문 맨 위 `> **📌 핵심 요약**` | ⚠️ 권장 |
| 목차 | 2000자+ 글에 `## Table of contents` | 💡 선택 |
| 섹션 구분 | 명확한 `##` 제목 구분 | ⚠️ 가독성 |
| 결론/마치며 | 마지막 섹션에 결론 있는가 | ⚠️ 독자 경험 |

### 3. SEO 검토

| 항목 | 기준 |
|------|------|
| title 길이 | 30~60자 권장 |
| description 길이 | 50~160자 |
| description 내용 | 클릭을 유도하는 구체적 가치 제안 |
| ogImage | 있으면 ✅, 없으면 자동생성(흑백) |

### 4. 기술 검토

| 항목 | 확인 방법 |
|------|----------|
| 이미지 경로 | `src/data/blog/images/` 기준 상대 경로 유효성 |
| 외부 링크 | 마크다운 링크 문법 오류 여부 |
| 코드 블록 | ` ``` ` 닫힘 여부 |

## 판정 기준

- 🚨 **발행 블로커** — 이 상태로 발행 시 오류나 심각한 SEO 손실 발생
- ⚠️ **개선 권장** — 발행은 가능하지만 품질에 영향
- 💡 **선택적 개선** — 있으면 좋지만 없어도 무방

---

## 출력 형식

```markdown
## [파일명] 검토 결과

### frontmatter
- ✅ title: "제목 내용"
- ❌ description: 누락 🚨
- ⚠️ ogImage: 없음 (SNS 공유 시 자동생성 이미지 사용)

### 구조
- ✅ 핵심 요약 박스: 존재
- ⚠️ 목차: 없음 (본문 2800자 — 추가 권장)

### SEO
- ✅ title 길이: 28자
- ❌ description: 없음 🚨

### 기술
- ✅ 이미지 경로: `./images/tokyo.jpg` 정상
- ✅ 코드 블록: 이상 없음
```
