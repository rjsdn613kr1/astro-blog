# 블로그 사용법 완벽 가이드

## 전체 구조 한눈에 보기

```
astro-blog/
├── src/
│   ├── data/blog/        ← ✅ 글 쓰는 곳 (Obsidian에서 여기에 작성)
│   ├── pages/about.md   ← ✅ 소개 페이지
│   └── config.ts        ← ⚙️  블로그 기본 설정 (VSCode로 수정)
├── templates/새글.md     ← 글 자동 템플릿 (건드리지 말 것)
├── 블로그_사용법.md      ← 이 파일 (블로그에 안 올라감)
└── .github/workflows/   ← 자동 배포 설정 (건드리지 말 것)
```

**블로그 주소**: https://rjsdn613kr1.github.io/astro-blog/
**GitHub 저장소**: https://github.com/rjsdn613kr1/astro-blog

## 글 쓰는 법

### 1단계. Obsidian에서 새 파일 만들기

왼쪽 파일 목록에서 `src` → `data` → `blog` 폴더 안에 새 파일 생성.

파일명은 영어 권장 (URL에 그대로 들어가기 때문)
예: `my-first-post.md`, `2026-investment-review.md`

### 2단계. 자동으로 채워지는 frontmatter 확인

Templater 플러그인 덕분에 파일을 만들면 아래가 자동 입력됩니다:

```
---
title: 파일명
author: suko_yasa
pubDatetime: 2026-04-02T10:30:00+09:00  ← 현재 시각 자동 입력
description:
tags:
  -
draft: false
---
```

**직접 채워야 하는 항목:**

| 항목 | 설명 | 예시 |
|------|------|------|
| `title` | 글 제목 | `나의 첫 번째 글` |
| `description` | 글 요약 (목록에 표시됨) | `오늘 있었던 일` |
| `tags` | 태그 (여러 개 가능) | `에세이`, `일상` |
| `category` | 카테고리 (하나만) | `에세이` |
| `draft: true` | 비공개 (블로그에 안 올라감) | |
| `draft: false` | 공개 | |

### 3단계. 본문 작성 후 저장

저장하면 Obsidian Git이 5분마다 자동으로 GitHub에 올립니다.
GitHub에 올라가면 2~3분 후 블로그에 반영됩니다.

**총 소요시간: 최대 8분**

### 즉시 배포하고 싶을 때
`Cmd+P` → `commit-and-sync` 실행

## Categories vs Tags 차이

| | Category | Tag |
|--|---------|-----|
| 개수 | 글 하나당 하나만 | 여러 개 가능 |
| 용도 | 큰 분류 | 세부 분류 |
| 예시 | `에세이` | `일상`, `감정`, `회고` |

**Category 사용 예시:**

```
---
title: 요즘 드는 생각
category: 에세이
tags:
  - 일상
  - 감정
---
```

→ 상단 네비게이션 **Categories** 클릭하면 카테고리별로 모아볼 수 있음
→ **Tags** 클릭하면 태그별로 모아볼 수 있음

## 상단 네비게이션 설명

| 메뉴 | 설명 |
|------|------|
| **Posts** | 전체 글 목록 |
| **Categories** | 카테고리별 글 목록 |
| **Tags** | 태그별 글 목록 |
| **About** | 소개 페이지 |
| 📁 아이콘 | 날짜별 아카이브 |
| 🔍 아이콘 | 글 검색 |

## 배포 흐름 (자동)

```
Obsidian에서 글 작성
      ↓ (5분마다 자동 or Cmd+P → commit-and-sync)
GitHub에 자동 업로드
      ↓ (약 2~3분)
블로그 자동 업데이트 완료
```

배포 상태 확인: https://github.com/rjsdn613kr1/astro-blog/actions
- 초록색 ✅ = 배포 완료
- 노란색 🟡 = 배포 중
- 빨간색 ❌ = 오류 발생

## 블로그 기본 설정 변경

`src/config.ts` 파일을 VSCode로 열어서 수정합니다.

| 항목 | 설명 | 현재 값 |
|------|------|---------|
| `title` | 블로그 제목 | `suko_yasa` |
| `author` | 작성자 이름 | `suko_yasa` |
| `desc` | 블로그 설명 | `생각과 경험을 기록하는 공간` |
| `postPerIndex` | 홈에 표시할 글 수 | `4` |

수정 후 `Cmd+P` → `commit-and-sync` 하면 배포됩니다.

## 문제 해결

| 증상 | 해결법 |
|------|--------|
| 글이 8분 지나도 안 보임 | `draft: false` 인지 확인 |
| Categories에 카테고리가 안 보임 | frontmatter에 `category:` 추가했는지 확인 |
| 배포 실패 메일 옴 | https://github.com/rjsdn613kr1/astro-blog/actions 에서 오류 확인 |
| Obsidian Git 작동 안 함 | `Cmd+P` → `commit-and-sync` 수동 실행 |
| 날짜가 이상하게 표시됨 | `pubDatetime` 형식 확인 (`2026-04-02T10:00:00+09:00`) |
