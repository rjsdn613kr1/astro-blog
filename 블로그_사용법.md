# ⚡ 처음 설정 (딱 한 번만 하면 됨)

새 볼트(astro-blog)에 플러그인을 새로 설치해야 합니다.

## 1단계: Obsidian Git 설치 (자동 업로드)

1. Obsidian 설정(⚙️) → **커뮤니티 플러그인**
2. **제한 모드 끄기** 클릭
3. **탐색** 버튼 → `Obsidian Git` 검색 → 설치 → 활성화
4. 설정 → Obsidian Git → **Auto commit-and-sync interval: `5`** 입력

→ 이제 5분마다 자동으로 GitHub에 업로드됩니다.

---

## 2단계: Templater 설치 (frontmatter 자동 입력)

1. **탐색** 버튼 → `Templater` 검색 → 설치 → 활성화
2. 설정 → **Templater**
3. **Template folder location** 항목에 `templates` 입력
4. **Trigger Templater on new file creation** → **켜기**
5. 아래로 스크롤 → **Folder Templates** → `+` 버튼
   - Folder: `src/data/blog`
   - Template: `templates/새글`

→ 이제 `src/data/blog` 안에 새 파일을 만들면 frontmatter가 자동으로 채워집니다.

# 글 쓰는 전체 흐름

```
① Obsidian에서 src/data/blog/ 안에 새 파일 생성
② frontmatter 자동 채워짐 → title, description, category, tags 입력
③ 본문 작성 → Cmd+S 저장
④ 최대 8분 후 블로그 반영
   (빨리 올리려면: Cmd+P → commit-and-sync)
```

# 글 쓰는 법

## 파일 만들기

Obsidian 왼쪽 파일 목록에서:
`src` → `data` → `blog` 폴더에 **우클릭 → 새 노트**

**파일명 규칙 (중요!)**
- 영어로 쓰기 (한글은 URL에서 깨질 수 있음)
- 띄어쓰기 대신 `-` 사용

| 좋은 예 | 나쁜 예 |
|---------|---------|
| `tokyo-trip.md` | `도쿄 여행.md` |
| `2026-book-review.md` | `독서감상문.md` |

---

## frontmatter 작성법

파일을 만들면 아래가 **자동으로** 채워집니다 (Templater 설정 후):

```
---
title: tokyo-trip          ← 파일명이 자동으로 들어감
author: suko_yasa
pubDatetime: 2026-04-02T14:30:00+09:00   ← 현재 시각 자동
modDatetime:               ← 수정 시 자동 업데이트
description: ""
tags: []
draft: false
---
```

**수정해야 하는 항목:**

| 항목 | 설명 | 예시 |
|------|------|------|
| `title` | 블로그에 표시되는 제목 | `도쿄 여행 3박 4일` |
| `description` | 글 목록에 나오는 한 줄 요약 | `처음 가본 도쿄 여행기` |
| `category` | 큰 분류 (하나만) | `에세이` |
| `tags` | 세부 태그 (여러 개 가능) | 아래 참고 |
| `draft` | `false`=공개, `true`=비공개 | |
| `modDatetime` | 최종 수정일 (수정 시 자동 갱신) | 자동 입력됨 |
| `featured` | `true`=홈 상단 Featured에 고정 | `featured: true` |
| `ogImage` | 메인 카드 썸네일 + SNS 공유 이미지 | 아래 참고 |

**완성 예시:**
```
---
title: 도쿄 여행 3박 4일
author: suko_yasa
pubDatetime: 2026-04-02T14:30:00+09:00
modDatetime: 2026-04-03T10:00:00+09:00
description: 처음 가본 도쿄, 먹고 걷고 사진 찍은 기록
category: 에세이
tags:
  - 여행
  - 일본
featured: false
draft: false
---

여기서부터 본문을 씁니다.
```

## 홈페이지 상단에 고정하고 싶은 글 (Featured)

frontmatter에 `featured: true` 추가:
```
---
title: 내 대표 글
featured: true
draft: false
---
```
→ 홈페이지 최상단 **"Featured"** 섹션에 표시됩니다.

**특징:**
- Featured 글이 1개 이상 있으면 홈에 "Featured" 섹션이 생깁니다
- 여러 개를 `featured: true`로 설정 가능
- `featured: false` 또는 항목 삭제 시 일반 "Recent Posts"로 이동

## 최종 수정일 표시하기 (modDatetime)

글을 수정하면 블로그에 **"Updated: 날짜"** 가 자동으로 표시됩니다.

### 자동 업데이트 (Claude Code 사용 시)

Claude Code로 블로그 파일을 수정하면 `modDatetime`이 **자동으로** 현재 시각으로 갱신됩니다. 따로 입력할 필요가 없습니다.

### 수동으로 설정하는 법

Obsidian에서 직접 수정할 경우, frontmatter에 수정 시각을 직접 입력:

```
---
title: 내 글
pubDatetime: 2026-04-01T10:00:00+09:00
modDatetime: 2026-04-03T15:30:00+09:00
---
```

→ 블로그에서 "Updated: 3 Apr, 2026"으로 표시됩니다.

**참고:** `modDatetime`이 `pubDatetime`보다 나중 날짜여야 "Updated:" 표시가 나타납니다. 같거나 이전 날짜면 작성일만 표시됩니다.

## 썸네일 설정 (ogImage)

`ogImage`는 두 가지 역할을 합니다:
1. **메인 페이지 카드 썸네일** — 글 목록에서 제목 옆에 작게 표시되는 미리보기 이미지
2. **SNS/메신저 공유 시 미리보기** — 카카오톡, Discord, Twitter 등에 링크 공유 시 나타나는 이미지

`ogImage`를 설정하지 않으면 카드에 이미지가 없고, SNS 공유 시 자동 생성된 흑백 텍스트 이미지가 사용됩니다.

### 방법 1: 외부 URL 사용 (빠르고 간편)

Unsplash 등에서 이미지 URL을 복사해서 바로 사용:

```
---
title: 도쿄 여행 3박 4일
ogImage: "https://images.unsplash.com/photo-xxxxx?w=1200"
---
```

### 방법 2: 로컬 이미지 사용

1. 썸네일 이미지를 Obsidian 글 편집 화면에 드래그 앤 드롭
   → `src/data/blog/images/` 에 자동 저장됨
2. frontmatter에 경로 입력:

```
---
title: 도쿄 여행 3박 4일
ogImage: ./images/tokyo-thumbnail.jpg
---
```

### 이미지 크기 권장값

| 항목 | 권장값 |
|------|--------|
| 크기 | 1200 × 630px |
| 형식 | PNG 또는 JPG |

**예시 frontmatter (모든 옵션 포함):**
```
---
title: 도쿄 여행 3박 4일
author: suko_yasa
pubDatetime: 2026-04-02T14:30:00+09:00
modDatetime:
description: 처음 가본 도쿄, 먹고 걷고 사진 찍은 기록
category: 에세이
tags:
  - 여행
  - 일본
featured: false
ogImage: "https://images.unsplash.com/photo-xxxxx?w=1200"
draft: false
---
```

## 핵심 요약 박스 만들기

레퍼런스 블로그처럼 본문 맨 위에 핵심 내용을 요약한 박스를 넣을 수 있습니다.

**방법:** 인용구(`>`) 문법으로 작성합니다.

```
> **📌 핵심 요약**
>
> - 요점 1
> - 요점 2
> - 요점 3
```

**실제 글에 배치하는 예시:**

```
---
title: AI로 일하는 법
---

> **📌 핵심 요약**
>
> - AI는 도구가 아닌 파트너로 써야 한다
> - 원리를 이해해야 AI를 제대로 활용할 수 있다
> - AI Native 엔지니어는 문제 정의가 핵심 역량이다

## 들어가며

본문 내용...
```

이모지 없이도 됩니다:
```
> **핵심 요약**
>
> 이 글의 핵심: 원리 없는 감각은 추측이다.
> AI 시대에는 '무엇을 묻느냐'가 실력의 척도가 된다.
```

## 목차 자동 생성 (Table of Contents)

글 안에 `## Table of contents` 제목을 쓰면, 그 아래에 자동으로 목차가 생성됩니다.

```
---
title: 긴 글 제목
---

## Table of contents

## 1장. 들어가며

내용...

## 2장. 본론

내용...

## 3장. 마치며

내용...
```

→ "Table of contents" 아래에 자동으로 목차 링크가 생성됩니다.

**접이식 목차 (기본 동작):** 목차가 자동으로 `▶ Table of contents` 접이식 박스로 감싸집니다. 클릭하면 펼쳐집니다.

# 문제 해결

| 증상 | 해결법 |
|------|--------|
| 새 파일 만들어도 frontmatter 안 나옴 | 위 **처음 설정** 의 Templater 설치 과정 다시 확인 |
| 글이 8분 지나도 안 보임 | `draft: false` 인지 확인 |
| 자동 업로드 안 됨 | Obsidian Git 설치 확인, 또는 `Cmd+P` → `commit-and-sync` 수동 실행 |
| 이미지가 블로그에서 안 보임 | 드래그 앤 드롭으로 다시 첨부 |
| 배포 실패 메일 옴 | actions 페이지에서 오류 확인 |
| 블로그 링크 404 뜸 | 2~3분 기다린 후 새로고침 |
| 카드 썸네일이 안 보임 | frontmatter에 `ogImage` 설정 (위 참고) |
| "Updated:" 가 안 나옴 | `modDatetime`이 `pubDatetime`보다 나중인지 확인 |
