# 블로그 사용법 완벽 가이드

## 전체 구조 한눈에 보기

```
astro-blog/
├── src/
│   ├── data/blog/        ← ✅ 글 쓰는 곳
│   ├── pages/about.md   ← ✅ 소개 페이지
│   └── config.ts        ← ⚙️  블로그 기본 설정
├── 블로그_사용법.md      ← 이 파일 (블로그에 안 올라감)
└── .github/workflows/   ← 자동 배포 설정 (건드리지 말 것)
```

**블로그 주소**: https://rjsdn613kr1.github.io/astro-blog/
**GitHub 저장소**: https://github.com/rjsdn613kr1/astro-blog

## 글 쓰는 법 (가장 중요)

### 1단계. Obsidian에서 새 파일 만들기

Obsidian 왼쪽 파일 목록에서
`src` → `data` → `blog` 폴더 안에 새 파일을 만드세요.

파일명은 영어로 쓰는 걸 권장합니다. (URL에 그대로 들어가기 때문)
예: `my-first-post.md`, `2026-investment-review.md`

### 2단계. 파일 맨 위에 정보 입력하기

파일을 열면 맨 위에 아래처럼 작성하세요. 이걸 **frontmatter**라고 합니다.

```
---
title: 여기에 글 제목을 씁니다
author: suko_yasa
pubDatetime: 2026-04-02T10:00:00+09:00
description: 글 목록에 보이는 짧은 요약문입니다
tags:
  - 에세이
  - 일상
draft: false
---

여기서부터 본문을 씁니다.
```

| 항목 | 설명 | 예시 |
|------|------|------|
| `title` | 글 제목 | `나의 첫 번째 글` |
| `pubDatetime` | 작성 날짜+시간 | `2026-04-02T10:00:00+09:00` |
| `description` | 글 요약 (목록에 표시됨) | `오늘 있었던 일을 기록했습니다` |
| `tags` | 태그 (여러 개 가능) | `에세이`, `일상`, `스터디` |
| `draft: true` | 비공개 | 블로그에 안 올라감 |
| `draft: false` | 공개 | 블로그에 올라감 |

### 3단계. 저장하면 자동 배포

파일 저장 후 Obsidian Git이 5분마다 자동으로 GitHub에 올립니다.
GitHub에 올라가면 약 2~3분 후 블로그에 반영됩니다.

**총 소요시간: 최대 8분**

### 즉시 배포하고 싶을 때

`Cmd+P` → `commit-and-sync` 입력 → 실행

---

## 날짜 형식 쉽게 쓰는 법

날짜 형식이 복잡해 보이지만 아래 형식만 외우면 됩니다.

```
pubDatetime: 2026-04-02T10:00:00+09:00
                년-월-일T시:분:초+09:00(한국시간)
```

예시:
- 2026년 4월 2일 오전 10시 → `2026-04-02T10:00:00+09:00`
- 2026년 12월 25일 오후 3시 → `2026-12-25T15:00:00+09:00`

---

## 카테고리(태그) 사용법

태그를 쓰면 자동으로 카테고리처럼 묶입니다.

```
tags:
  - 에세이
  - 일상
```

여러 글에 같은 태그를 쓰면 블로그에서 태그별로 모아볼 수 있습니다.

---

## 배포 흐름 (자동)

```
Obsidian에서 글 작성
      ↓ (5분마다 자동 or Cmd+P → commit-and-sync)
GitHub에 자동 업로드
      ↓ (약 2~3분)
블로그 자동 업데이트
```

배포 상태 확인: https://github.com/rjsdn613kr1/astro-blog/actions
- 초록색 ✅ = 배포 완료
- 노란색 🟡 = 배포 중
- 빨간색 ❌ = 오류 발생

## 소개 페이지(About) 수정하는 법

`src/pages/about.md` 파일을 열어서 수정하면 됩니다.
Obsidian에서 바로 편집 가능합니다.

---

## 블로그 기본 설정 변경하는 법

`src/config.ts` 파일을 VSCode로 열어서 수정합니다.
(Obsidian이 아닌 VSCode에서 수정해야 합니다)

| 항목 | 설명 | 현재 값 |
|------|------|---------|
| `title` | 블로그 제목 | `suko_yasa` |
| `author` | 작성자 이름 | `suko_yasa` |
| `desc` | 블로그 설명 | `생각과 경험을 기록하는 공간` |
| `postPerIndex` | 홈에 표시할 글 수 | `4` |

수정 후 Obsidian에서 `Cmd+P` → `commit-and-sync` 하면 배포됩니다.

## 문제 해결

| 증상 | 해결법 |
|------|--------|
| 글이 8분 지나도 안 보임 | `draft: false` 인지 확인 |
| 날짜가 이상하게 표시됨 | `pubDatetime` 형식 확인 |
| 배포 실패 | https://github.com/rjsdn613kr1/astro-blog/actions 에서 오류 확인 |
| Obsidian Git 작동 안 함 | `Cmd+P` → `commit-and-sync` 수동 실행 |
