## 1. 기본 개념 (먼저 읽기)

**글을 쓰면 블로그에 올라가는 흐름:**
```
Obsidian에서 글 작성 (내 컴퓨터)
        ↓ 자동 (최대 5분)
GitHub 저장소에 업로드
        ↓ 자동 (약 2~3분)
블로그에 반영
```
**총 대기시간: 최대 8분**

즉시 올리고 싶으면: `Cmd+P` → `commit-and-sync` 실행

## 2. 글 쓰는 법

### 글 파일 만들기
Obsidian 왼쪽 파일 목록에서
`src` → `data` → `blog` 폴더를 찾아서 거기에 새 파일 만들기

> **파일명 규칙**
> - 영어로 쓰기 (URL에 그대로 들어감)
> - 띄어쓰기 대신 `-` 사용
> - 예: `my-first-post.md`, `2026-trip-to-japan.md`

### 파일 맨 위에 정보 입력하기 (frontmatter)
파일을 만들면 아래 내용이 **자동으로** 채워집니다 (Templater 플러그인):

```
---
title: 파일명
author: suko_yasa
pubDatetime: 2026-04-02T14:30:00+09:00
description:
tags:
  -
draft: false
---
```

**직접 수정해야 하는 부분:**

| 항목 | 설명 | 예시 |
|------|------|------|
| `title` | 글 제목 (블로그에 표시됨) | `도쿄 여행기` |
| `description` | 글 요약 (목록에 표시됨) | `3박 4일 도쿄 여행 기록` |
| `tags` | 태그. `-` 다음에 태그 입력 | `- 여행` |
| `category` | 카테고리 (하나만) | `에세이` |
| `draft` | `false`면 공개, `true`면 비공개 | |

**완성된 예시:**
```
---
title: 도쿄 여행기
author: suko_yasa
pubDatetime: 2026-04-02T14:30:00+09:00
description: 3박 4일 도쿄 여행 기록
tags:
  - 여행
  - 일본
category: 에세이
draft: false
---

여기서부터 본문을 씁니다.
```

### 본문 작성 후 저장
`Cmd+S` 로 저장하면 끝. 나머지는 자동.

## 4. 사진/이미지 첨부하는 법

### 방법 1: 드래그 앤 드롭 (가장 쉬움)
1. 사진 파일을 Obsidian 글 편집 화면에 그냥 **드래그해서 놓기**
2. 자동으로 아래처럼 삽입됨:
   ```
   ![[사진파일명.jpg]]
   ```
3. 이미지 파일은 자동으로 `src/data/blog/` 폴더에 복사됨

### 방법 2: 복사 붙여넣기
1. 사진을 `Cmd+C` 로 복사
2. Obsidian 글 편집 화면에서 `Cmd+V` 로 붙여넣기
3. 자동으로 삽입됨

### 방법 3: 직접 입력
```
![사진 설명](사진파일명.jpg)
```
- 사진 파일을 `src/data/blog/` 폴더 안에 직접 넣은 경우
- `사진 설명` 부분은 사진이 안 보일 때 표시되는 텍스트

> **팁:** 방법 1 (드래그)이 제일 편합니다. 그냥 끌어다 놓으면 됩니다.

## 5. 카테고리와 태그 사용법

### 카테고리 (큰 분류)
- 글 하나당 **하나만** 지정
- frontmatter에 `category:` 로 입력
- 홈페이지에 카테고리별 섹션으로 자동 표시됨

```
category: 에세이
```

### 태그 (세부 분류)
- **여러 개** 가능
- frontmatter에 `tags:` 아래 `- 태그명` 형식으로 입력

```
tags:
  - 일상
  - 감정
  - 2026
```

### 폴더로 분류하기
`src/data/blog/` 안에 폴더를 만들어도 됩니다.
```
blog/
  에세이/
    첫번째글.md
    두번째글.md
  스터디/
    공부기록.md
```

## 6. 비공개 글 작성법

### 임시저장 (draft)
frontmatter에서 `draft: true` 로 바꾸면 블로그에 안 올라갑니다.
나중에 `draft: false` 로 바꾸면 공개됩니다.

```
draft: true   ← 비공개
draft: false  ← 공개
```

### 완전 비공개 폴더
`src/data/blog/private/` 폴더를 만들고 거기에 넣으면
GitHub에는 올라가지만 블로그에는 절대 표시되지 않습니다.

## 7. 글 수정/삭제하는 법

### 수정
Obsidian에서 파일 열고 수정 후 저장 → 자동 배포

### 삭제
Obsidian에서 파일 선택 후 우클릭 → 삭제 → 자동 배포

## 9. 블로그 설정 변경 (제목, 설명 등)

`src/config.ts` 파일을 VSCode(또는 텍스트 편집기)로 열어서 수정합니다.
(Obsidian으로 수정하면 안 됨)

| 항목 | 설명 | 현재 값 |
|------|------|---------|
| `title` | 블로그 제목 | `suko_yasa` |
| `author` | 작성자 이름 | `suko_yasa` |
| `desc` | 블로그 설명 | `생각과 경험을 기록하는 공간` |
| `postPerIndex` | 홈에 표시할 글 수 | `4` |

수정 후 Obsidian에서 `Cmd+P` → `commit-and-sync` 하면 배포됩니다.

---

## 10. 즉시 배포하는 법

`Cmd+P` → `commit-and-sync` 입력 후 실행

배포 진행 상황: https://github.com/rjsdn613kr1/astro-blog/actions
- ✅ 초록색 = 완료
- 🟡 노란색 = 진행 중 (2~3분 기다리기)
- ❌ 빨간색 = 오류 (저한테 물어보기)

---

## 11. 문제 해결

| 증상 | 해결법 |
|------|--------|
| 글이 8분 지나도 블로그에 안 보임 | frontmatter에 `draft: false` 인지 확인 |
| 이미지가 블로그에서 안 보임 | 이미지 파일이 `src/data/blog/` 폴더 안에 있는지 확인 |
| 배포 실패 메일 옴 | actions 페이지에서 오류 내용 확인 후 저한테 문의 |
| Obsidian Git 작동 안 함 | `Cmd+P` → `commit-and-sync` 수동 실행 |
| 블로그 링크 클릭하면 404 뜸 | 저한테 문의 |
