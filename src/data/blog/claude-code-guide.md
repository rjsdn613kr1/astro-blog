## Claude Code가 뭔가

ChatGPT나 Claude.ai는 웹 브라우저에서 쓰는 채팅 인터페이스다. Claude Code는 다르다. **터미널에서 실행하는 CLI 도구**로, Claude가 내 컴퓨터의 파일을 직접 읽고 수정할 수 있다.

쉽게 말하면:

> "이 블로그의 포스트 레이아웃에 최종 수정일이 표시되게 해줘"

라고 말하면 Claude가 알아서 관련 파일을 찾아서 코드를 수정해준다. 내가 직접 코드를 짤 필요가 없다.

개발자가 아닌 사람도 쓸 수 있다. 코드를 몰라도 "어떻게 해줘"라고 말하면 된다.

## 설치하는 법

Node.js가 설치되어 있어야 한다. 터미널을 열고:

```bash
npm install -g @anthropic-ai/claude-code
```

설치 후 프로젝트 폴더로 이동해서 실행:

```bash
cd 내-프로젝트-폴더
claude
```

처음 실행하면 Anthropic 계정으로 로그인하라고 한다. 로그인하면 바로 사용 가능.

## 실제 사용 흐름

Claude Code를 실행하면 대화창이 뜬다. 여기서 자연어로 지시하면 된다.

### 파일 수정 요청

```
나: 블로그 포스트에 최종 수정일이 안 보이는데 수정해줘
Claude: PostDetails.astro 파일을 확인하겠습니다... (파일 읽음)
        modDatetime 값이 있을 때 "Updated:" 표시가 되도록 이미 구현되어 있네요.
        frontmatter에 modDatetime 필드가 없어서 그런 것 같습니다.
        템플릿 파일을 수정할까요?
나: 응, 수정해줘
Claude: templates/new.md를 수정했습니다. 앞으로 새 글을 만들면 modDatetime 필드가 자동으로 포함됩니다.
```

이게 전부다. Claude가 어떤 파일을 봐야 하는지 스스로 파악하고, 수정하고, 결과를 알려준다.

### 새 기능 추가 요청

```
나: 글을 수정할 때마다 modDatetime이 자동으로 업데이트되게 해줘
Claude: .claude/hooks/ 폴더에 스크립트를 만들고,
        settings.json에 hook을 등록하겠습니다...
```

## 알아두면 좋은 개념

### 퍼미션 (Permission)

Claude Code는 파일 수정, 명령어 실행 같은 작업을 하기 전에 **허락을 구한다**. 한 번 허락하면 그 세션 동안은 계속 허용된다.

자주 쓰는 명령어는 `.claude/settings.json`에 미리 허용해두면 매번 물어보지 않는다:

```json
{
  "permissions": {
    "allow": ["Bash(git commit*)", "Bash(npm run build)"]
  }
}
```

### 훅 (Hook)

특정 동작이 일어났을 때 자동으로 실행되는 스크립트. 예를 들어:

- 파일을 수정할 때마다 `modDatetime` 자동 갱신
- 빌드 전에 자동으로 lint 실행

이 블로그에서는 블로그 글(`.md` 파일)이 수정될 때마다 자동으로 최종 수정일을 현재 시각으로 갱신하는 훅을 설정했다.

### 하네스 (Harness)

프로젝트에 맞는 AI 에이전트와 스킬을 미리 구성해두는 것. 예를 들어 이 블로그에는:

- **blog-writer 에이전트**: 새 블로그 글 작성 전문
- **blog-reviewer 에이전트**: 발행 전 품질 검토 전문
- **write-post 스킬**: 이 블로그의 frontmatter 규칙, 글 구조 가이드 포함

한 번 설정해두면 다음부터 "글 써줘"라고만 해도 이 프로젝트의 규칙에 맞게 알아서 작성한다.

## 이 블로그에서 실제로 쓰는 방식

이 블로그(suko_yasa)는 Obsidian으로 글을 쓰고, Claude Code로 블로그 자체를 관리한다.

**Obsidian이 하는 일:**
- 새 글 작성
- 이미지 드래그 앤 드롭
- 5분마다 자동 GitHub 업로드

**Claude Code가 하는 일:**
- 블로그 레이아웃/기능 수정
- 사용법 가이드 업데이트
- 자동화 스크립트 설정
- 새 블로그 글 작성 (테스트 겸)

이 글도 Claude Code가 작성했다.

## 유용한 팁

**1. 작업 전 파일 경로를 알려주면 빠르다**

```
"src/layouts/PostDetails.astro 파일에서 태그 표시 부분을 수정해줘"
```

**2. 한 번에 여러 작업을 묶어서 요청해도 된다**

```
"블로그 포스트 템플릿에 modDatetime 추가하고,
 사용법 가이드에도 설명 추가하고,
 hook 스크립트도 만들어줘"
```

**3. 모르면 물어보면 된다**

```
"이 프로젝트 구조가 어떻게 되어 있어?"
"이 오류는 왜 나는 거야?"
"이걸 어떻게 하는 게 제일 좋아?"
```

Claude가 코드베이스를 읽고 설명해준다.

## 마치며

Claude Code의 핵심은 **대화**다. "어떻게 해줘"라고 말하면 Claude가 알아서 방법을 찾는다. 코드를 직접 짤 필요도, 문서를 뒤질 필요도 없다.

처음엔 어색할 수 있다. 어디까지 요청할 수 있는지 감이 안 잡히기도 한다. 하지만 써볼수록 "이것도 되네?"라는 순간이 늘어난다.

이 블로그의 거의 모든 기능 업데이트는 Claude Code로 이루어졌다. 직접 써보는 게 가장 빠른 이해다.
